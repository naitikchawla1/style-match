[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's take a look at the system architecture. When a user submits a styling request, it first enters the security checkpoint node. This node scrubs any personal identifiable information like emails or phone numbers, detects prompt injections, and logs metadata. If everything is clean, the request travels to our main orchestrator agent. The orchestrator delegates to two specialized sub-agents: the wardrobe agent, which calls the local Model Context Protocol server to fetch the user's clothes, and the weather matcher agent, which queries the MCP server for the local forecast. Once they return suggestions, the orchestrator combines them and sends the output to the outfit approval checkpoint. Here, we pause the workflow to wait for human input. If the user approves, it routes to final output and saves the selection to their diary; if not, it loops back to the orchestrator to revise the choices based on user feedback.
